# 迁移

迁移不必从整个应用开始。选择一个职责清楚的页面区域，把行为、样式和验证一次闭环，再将同样的步骤推广到下一个区域。

## 迁移顺序

先盘点当前交互与可访问性要求，然后安装 MoriUI 并接入全局样式。优先替换 Button、Input 等叶子组件，最后处理 Dialog、菜单和导航这类需要上下文的组合组件。

## 替换样式

删除旧组件专属的颜色、圆角和焦点样式，改用 Nova 的语义 Token 工具类。不要把历史选择器搬进新组件，也不要在页面内复制主题变量。

## 验证交互

每替换一个区域，都要运行类型检查和测试，并用键盘确认焦点顺序、Escape 关闭、禁用状态与错误提示。浅色和深色模式都通过后，再删除旧实现。

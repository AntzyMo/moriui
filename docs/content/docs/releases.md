# 发布说明

发布记录关注会影响组件使用者的变更：公开属性、交互行为、样式入口与最低运行环境。修正文案或内部整理不会被包装成额外的迁移步骤。

## 版本策略

版本号遵循语义化版本。补丁版本修复兼容问题，次版本加入向后兼容的能力，主版本才允许调整现有公开契约。处于预发布阶段的能力会在说明中单独标注。

## 升级前

先查看当前项目使用的组件，再核对目标版本的变更范围。提交锁文件之前，确认 Vue、Tailwind CSS 与构建工具满足新的最低版本要求。

## 升级后

运行类型检查、组件测试与静态构建，并用键盘复查 Dialog、Sheet 和菜单等浮层组件。若视觉结果变化，优先检查是否仍只加载一个 `moriui/style.css` 入口。
